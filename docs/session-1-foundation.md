# Сессия 1 — Фундамент (~35%)

**Цель:** рабочий каркас, на котором можно строить фичи. К концу сессии —
приложение запускается, можно залогиниться, роли работают, навигация есть.

## 1. Инфраструктура проекта

- [ ] Создать Flutter-проект, подключить Firebase (Auth, Firestore, Storage, Functions)
- [ ] Зафиксировать стек: Riverpod, go_router, DI, freezed
- [ ] Структура папок (core / features / shared / data / domain / presentation)
- [ ] Firebase Emulator Suite для локальной разработки
- [ ] Тема приложения (светлая/тёмная), базовые переиспользуемые виджеты

## 2. Авторизация и роли

- [ ] Регистрация / вход по email+паролю
- [ ] Восстановление пароля, подтверждение email
- [ ] Модель пользователя + профиль (коллекции `users`, `profiles`)
- [ ] 3 роли (админ/преподаватель/студент) + редирект на нужный интерфейс после входа
- [ ] Базовые Firestore Security Rules (каркас, проверка ролей)

## 3. Навигация-скелет

- [ ] Боковое меню (web) / нижняя навигация (mobile), адаптивность
- [ ] Пустые экраны-заглушки для всех основных разделов
- [ ] Dashboard'ы для трёх ролей (пока без данных)

## Результат сессии

Приложение запускается на web/mobile, проходит регистрация и вход,
пользователь по роли попадает на свой dashboard, навигация между разделами
работает (с заглушками). Фундамент готов для наполнения фичами в Сессии 2.
