# Сессия 2 — Основная функциональность (~65%)

**Цель:** рабочее приложение по критериям приёмки. Идём по сущностям сверху вниз.

## 4. Админка

- [ ] Управление пользователями (создание преподавателей через Cloud Function,
      блокировка, назначение ролей)
- [ ] Управление группами и курсами (CRUD, назначения)

## 5. Курсы, группы, материалы

- [ ] Группы, курсы, недели курса (CRUD)
- [ ] Материалы (текст, ссылка, файл, YouTube-видео внутри приложения)
- [ ] Публикация/скрытие, доступ студентов к курсам

## 6. Задания и оценки

- [ ] Создание заданий, дедлайны
- [ ] Отправка студентом (текст/файл/ссылка)
- [ ] Проверка, выставление оценок, комментарии, журнал оценок

## 7. Банк вопросов и квизы ⭐ (самый объёмный блок, ~половина сессии)

- [ ] Банк вопросов + основные типы (single / multiple / true-false / текст —
      остальные при наличии времени)
- [ ] Импорт TXT с предпросмотром и валидацией
- [ ] Создание квиза, рандомайзер по категориям
- [ ] Прохождение квиза студентом, snapshot вопросов
- [ ] Расчёт баллов на сервере (Cloud Function)

## 8. Статистика, уведомления, безопасность, тесты

- [ ] Статистика студента / группы / курса
- [ ] Уведомления (in-app + push)
- [ ] Доработать и протестировать Security Rules
- [ ] Тестирование (web / Android / iOS), фиксы, релизные сборки

## MVP-приоритет (если время поджимает)

Блок 7 — кандидат №1 на сокращение: оставить 4 типа вопросов вместо 12.
Остальные блоки тоже резать до базового CRUD без «продвинутых» опций
